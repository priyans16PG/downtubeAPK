TubeGrab Full UI Reboot - Product Design Specification
======================================================

Goal
----
Redesign TubeGrab from a single-page utility interface into a modern multi-screen mobile application with a professional download manager experience similar to large consumer apps.

The design should feel structured, minimal, and confident, with clear task stages and dedicated screens instead of stacked cards.

Primary product flow:
Home -> Fetch -> Format Selection -> Download Progress -> Download Manager -> History

The application must feel like a polished consumer product rather than a developer tool.

Core Design Principles
----------------------
1) Task-first UI
Every screen should focus on a single task.

2) Minimal cognitive load
Avoid long scroll pages with mixed controls.

3) Strong visual hierarchy
Primary actions must be obvious and dominant.

4) Clear state transitions
User always understands what stage they are in.

5) Dedicated download management
Downloads must be visible and controllable at all times.

Application Structure
---------------------
The app will contain these main screens:
1) Home Screen
2) Media Info Screen
3) Format Selection Screen
4) Active Download Screen
5) Download Manager Screen
6) History Screen

Bottom navigation is used for quick access to Downloads and History.

SCREEN 1 - Home (URL Entry)
---------------------------
Purpose
This screen acts as the starting point for every action.

Layout
Top Navigation Bar
- Logo on left
- Title: TubeGrab
- Subtitle: Fast Video Downloader
- Version number small on the right

Main Input Section (center of screen)
- Large URL input field with placeholder: "Paste video or playlist link"
- Left icon: link symbol
- Right utility button: Paste

Primary Action Button
- Full width button: Fetch Info
- Must be visually strong

Helper Text
- "Supports single videos and playlists."

Recent Links Section
- Show recent URLs used under main input
- Each row: video title or shortened URL
- Tap row to reuse

SCREEN 2 - Media Information
----------------------------
Purpose
After fetching metadata, user is taken to a preview screen.

Layout
Top App Bar
- Back arrow
- Title: Video Info

Media Preview Card
- Thumbnail image
- Video title
- Channel name
- Duration

If playlist
- Playlist thumbnail
- Playlist title
- Number of videos

Playlist Mode Selector
- Segmented control
- Single Video
- First Video
- Full Playlist
- Default when playlist detected: Full Playlist

Bottom Action
- Continue button: "Choose Format"

SCREEN 3 - Format Selection
---------------------------
Purpose
Allow the user to choose download quality.

Layout
Top Navigation Bar
- Back arrow
- Title: Select Format

Video Info Summary (compact)
- Small thumbnail
- Title
- Duration

Formats List
- Scrollable list of available formats

Each row contains
- Radio indicator
- Format name (example: 1080p MP4)
- Optional file size
- Optional bitrate or resolution info

Example rows
- o 1080p Video (MP4) - 65MB
- o 720p Video (MP4) - 32MB
- o Audio Only (M4A) - 160kbps

Selected row must be highlighted.

Save Location Section
- Folder icon
- Download path
- Change button

Primary Action Button
- Start Download

SCREEN 4 - Active Download Screen
---------------------------------
Purpose
Display detailed progress and download controls.

Layout
Top Navigation Bar
- Title: Downloading

Download Card
- Thumbnail
- Video title

Large Progress Bar
- Percentage text centered above bar

Example
52%
[progress bar visual]

Detailed Metrics
- Downloaded size
- Total size
- Download speed
- Estimated time remaining

Example
64MB / 120MB
1.7MB/s | ETA 00:35

Control Buttons
- Pause
- Resume
- Cancel

Pause changes to Resume when paused.

Cancel behavior
- Stops download
- Moves item to failed history

Completion State
- Message: Download Finished
- Actions:
  - Open File
  - Show in Folder
  - Return to Home

SCREEN 5 - Download Manager
---------------------------
Purpose
Central location for all downloads.

Layout
Top Navigation Bar
- Title: Downloads

Tabs
- Active
- Completed
- Failed

Active Tab rows
- Thumbnail
- Title
- Progress bar
- Pause button
- Cancel button

Completed Tab rows
- Thumbnail
- Title
- File size
- Open button

Failed Tab rows
- Thumbnail/title/status
- Retry button

SCREEN 6 - History
------------------
Purpose
Shows the last downloads and actions.

Each row
- Time
- Status icon
- Title

Examples
- check 12:14 Download Completed
- cross 11:58 Download Failed

Overlay Components
------------------
Loading Overlay
- Displayed during fetch/process steps
- Centered spinner
- Message example: "Fetching video information..."

Splash Screen
- Centered logo
- App name
- Subtitle
- Bottom text: "Initializing downloader..."
- Dark gradient background

Design System
-------------
Spacing
- 8px micro spacing
- 16px component spacing
- 24px section spacing
- 32px screen spacing

Corner Radius
- Inputs: medium
- Cards: medium
- Buttons: medium

Buttons
- Primary: main actions
- Secondary: optional actions
- Danger: cancel/delete actions

Interaction Behavior
--------------------
Fetch Flow
- User pastes link
- Presses Fetch Info
- Loader appears
- Media Info screen opens

Download Flow
- User chooses format
- Presses Start Download
- Active Download screen opens
- User can pause, resume, or cancel

Completion Flow
- Success state appears
- File actions available

Visual Style
------------
- Dark theme background
- High-contrast primary text
- Muted secondary text
- Accent colors only for interactive states
- Avoid excessive borders
- Use spacing and contrast for separation

Important Constraints
---------------------
- Do not reuse the original stacked-card single-page layout.
- The app should feel like a modern mobile application with clear screens, structured navigation, and a professional download manager.
